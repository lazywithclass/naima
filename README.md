# Naima

NixOS EC2 instance with Docker and Claude Code Remote Control.
Fully autonomous after first deploy — no operator presence needed at runtime.

## Architecture

```
Your phone / browser
        |
        |  HTTPS (outbound from EC2 only)
        v
  Anthropic relay API
        |
        v
  EC2 (claude remote-control process)
        |
        +-- /srv/project  (cloned via sealed deploy key at boot)
        +-- Docker daemon
```

No inbound ports are opened for Claude Code. The relay is outbound-only.
SSH (port 22) is the only inbound port, and only for `colmena apply`.

OpenTofu provisions the AWS resources, then automatically seals credentials
and runs Colmena to configure the machine.

## Repository layout

### What lives in secrets/

| File | Safe to commit? | Notes |
|---|---|---|
| `instance-ip.txt` | No (gitignored) | Generated by tofu |
| `deploy-key.pub` | Yes | Generated by tofu, add to GitHub Deploy Keys |
| `deploy-key.cred` | Yes | Encrypted blob, machine-bound |
| `session-token.cred` | Yes | Encrypted blob, machine-bound |
| `repo-config.json` | No (gitignored) | Generated by tofu from variables |

## Prerequisites

`opentofu` and `colmena` are installed by `shell.nix`

``` bash
eval $(ssh-agent) && ssh-add ~/.ssh/id_ed25519
```
Subscription: Claude Code Remote Control requires a **Pro or Max** claude.ai plan.

## Deployment

### 1. Configure

```bash
cd tofu
cp terraform.tfvars.example terraform.tfvars
# Set: nixos_ami_id, ssh_public_key, repo_url
```

Find NixOS AMIs: https://nixos.org/download/#nixos-amazon

### 2. Deploy

```bash
tofu init && tofu apply
```

This provisions EC2, generates a deploy key, seals credentials on the
instance, and runs `colmena apply` automatically.

SSH access is restricted to your current public IP, detected automatically
via `ifconfig.me` at apply time. If your IP changes, run `tofu apply` again
to update the security group.

### 3. Add deploy key to GitHub

```bash
tofu output deploy_key_public
```

Copy the output and add it to:
GitHub -> repo -> Settings -> Deploy keys -> Add deploy key
Title: `naima` | Allow write access: **No**

Then restart the git-clone service (or wait for next boot):
```bash
ssh root@$(cat ../secrets/instance-ip.txt) systemctl restart git-clone-project
```

### 4. Activate Claude session

```bash
./scripts/activate-session.sh
```

This opens an interactive SSH session. Run `claude`, authenticate via browser,
then type `exit`. The script seals the real session token and restarts the service.

## After deploy: what happens at each boot

1. Network comes up
2. `git-clone-project` — clones/pulls the repo using the sealed deploy key
3. `claude-code-remote` — starts `claude remote-control`, registers session with Anthropic relay
4. Session appears in Claude app / claude.ai/code

## Updating secrets

Re-run `./scripts/activate-session.sh` to rotate the session token.
To rotate the deploy key, taint and re-apply:
```bash
cd tofu && tofu apply -replace=tls_private_key.deploy
```
Then update the deploy key on GitHub and run `activate-session.sh` again.

## Important limits

- **10-minute network timeout**: if the instance loses network for >10 minutes,
  the Claude Code session exits. `Restart=on-failure` in the systemd unit
  will restart it, but the session URL changes. Check `journalctl -u claude-code-remote`.
- **One remote session per process**: only one device can be actively controlling
  the session at a time.
- **Pro or Max plan required**: session token auth is required for Remote Control.
